Docker Creation¶

You can copy your existing state to docker /data directory adjusting for paths.

  • The UID/GID should be set to a user/group that has access to your media. All media clients should run under the same user to run correctly.

Create custom_server script to manage server, per 7 Days Docker and Documentation requirements.

0755 7days 7days /data/custom_server¶
#!/bin/bash
#
# Runs as root. Drop privileges.
#
# Capture kill/term signals and send SIGINT to gracefully shutdown 7days server.
PROCESS_WAIT_TIME=25
WATCHDOG_TIME=300

function shutdown() {
  echo 'Shutting down server ...'
  if [ "$(pgrep -n 7Days)" != '' ]; then
    echo "Sending SIGQUIT to 7 Days server (max ${PROCESS_WAIT_TIME} secs) ..."
    kill -SIGQUIT `pgrep -n 7Days`
    sleep ${PROCESS_WAIT_TIME}
  fi
  exit 0
}
trap shutdown SIGINT SIGKILL SIGTERM

function start_server() {
  su steam -c "/data/server/startserver.sh -configfile=/data/server/serverconfig.xml"
}

function watch_server() {
  if ps aux | grep [7]DaysToDieServer > /dev/null; then
    echo 'Server is running ...'
  else
    echo 'Starting server ...'
    start_server &
  fi
}

while true; do
  watch_server
  # background and using wait enables trap capture.
  sleep ${WATCHDOG_TIME} &
  wait
done

Note

  • PROCESS_WAIT_TIME: amount of time allocated to shutdown wine.

  • WATCHDOG_TIME: amount of time between checking server heartbeat.

Game options specified will launch a dedicated server visible on steam clients. The game server will check for updates on boot and reboot.

Initial Startup¶

7 Days needs to be started twice to generate the basic files in the correct locations for server configuration.

Start 7 Days server to generate configuration files.¶
docker-compose up -d 7days

Note

Wait until you see the message:

Using config file: /data/server/serverconfig.xml

This means that the server has started.

Stop 7 Days server and configure it.¶
docker-compose stop 7days
